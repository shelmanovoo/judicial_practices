```cypher
// Создаем узлы для судебных дел (кейсов)

(case8:Case {id: "88-17480/2020", court: "Третий КСОЮ", outcome: "проигрыш_собственника", description: "Требование вернуть открытие шлагбаума с пульта вместо звонка" }),

(principle1:LegalPrinciple {name: "Нельзя ограничивать право пользования общим имуществом" }),
(principle2:LegalPrinciple {name: "Решение собрания не может нарушать права собственников" }),
(principle3:LegalPrinciple {name: "Оплата установки не дает исключительных прав" }),
(principle4:LegalPrinciple {name: "Доступ должен быть у всех собственников" }),
(principle5:LegalPrinciple {name: "Ограничения для арендаторов законны" }),

(case2)-[:BASED_ON]->(principle2),
(case3)-[:BASED_ON]->(principle3),

(case4)-[:BASED_ON]->(principle1),

(case7)-[:BASED_ON]->(principle6),
(case8)-[:BASED_ON]->(principle6)
```

Этот граф будет содержать:

1. **Узлы Case** - судебные дела с атрибутами (номер, суд, исход, описание)
2. **Узлы LegalPrinciple** - правовые принципы, на которые ссылались суды
3. **Узлы Participant** - участники процессов (собственники, собрания, УК, арендаторы)
4. **Узлы Object** - объекты спора (шлагбаумы, пульты, парковки)
5. **Связи** между делами и участниками/объектами/принципами

Граф позволит анализировать:
- Какие дела выигрывают/проигрывают собственники
- Какие правовые принципы применяются в разных ситуациях
- Кто является участниками споров
- Какие объекты становятся предметом разбирательств
